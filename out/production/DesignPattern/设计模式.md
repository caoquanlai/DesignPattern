# 设计模式的目的

1. 可复用性

   相同功能的代码，不用重复多次编写

2. 可读性

   编码规范性，便于程序员阅读和理解

3. 可扩展性

   需要增加新的功能时，非常方便

4. 可靠性

   增加的新功能对原来的功能没有影响

5. 高内聚，低耦合

# 七大原则

1. 单一职责原则
2. 接口隔离原则
3. 依赖倒转（倒置）原则
4. 里氏替换原则
5. 开闭原则
6. 迪米特法则
7. 合成复用原则

#### 单一职责原则

​	对于类层面的原则，即一个类应该只负责一项职责。如类A负责两个不同的职责：职责1，职责2。当职责1需求变更而改变A时，可能造成职责2执行错误，所以需要将类A的粒度分解为A1，A2。
